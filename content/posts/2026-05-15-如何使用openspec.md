## 什么是 OpenSpec？

OpenSpec，通常指的是 **OpenAPI Specification**（原名 Swagger Specification），是一种用于描述 RESTful API 的标准化格式。它使用 YAML 或 JSON 来定义 API 的端点、请求参数、响应格式、认证方式等信息。

简单来说，OpenSpec 就是一份 **API 的说明书**——机器能读懂，人也能看明白。

## 为什么要用 OpenSpec？

在 AI 应用开发中，API 无处不在：调用大模型、访问向量数据库、对接各种微服务。OpenSpec 能帮你：

- **标准化文档**：团队成员不需要口头沟通 API 细节
- **自动生成代码**：客户端 SDK、服务端 stub 都能自动生成
- **API 测试**：配合 Swagger UI 直接在浏览器里测试接口
- **AI Agent 集成**：很多 AI Agent 框架使用 OpenSpec 来理解和调用外部工具

## 快速开始

### 1. 基本结构

一个最简单的 OpenSpec 文件长这样：

```yaml
openapi: 3.0.3
info:
  title: AI 学习笔记 API
  description: 管理 AI 学习笔记的 RESTful API
  version: 1.0.0

servers:
  - url: https://api.example.com/v1
    description: 生产环境

paths:
  /notes:
    get:
      summary: 获取所有学习笔记
      operationId: getNotes
      tags:
        - notes
      parameters:
        - name: category
          in: query
          description: 按分类筛选
          schema:
            type: string
      responses:
        '200':
          description: 成功返回笔记列表
          content:
            application/json:
              schema:
                type: array
                items:
                  $ref: '#/components/schemas/Note'

components:
  schemas:
    Note:
      type: object
      properties:
        id:
          type: integer
        title:
          type: string
        content:
          type: string
        category:
          type: string
        created_at:
          type: string
          format: date-time
```

### 2. 核心概念解析

| 字段 | 作用 |
|------|------|
| `openapi` | 指定规范版本（推荐 3.0.3 或 3.1.0） |
| `info` | API 的基本信息：标题、描述、版本号 |
| `servers` | API 服务器地址 |
| `paths` | 所有 API 端点的定义 |
| `components` | 可复用的数据模型（Schema）、参数、响应等 |

### 3. 定义请求体（POST 请求）

```yaml
paths:
  /notes:
    post:
      summary: 创建新笔记
      operationId: createNote
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              required:
                - title
                - content
              properties:
                title:
                  type: string
                  example: "Transformer 学习笔记"
                content:
                  type: string
                  example: "今天学习了 Self-Attention 机制..."
                tags:
                  type: array
                  items:
                    type: string
                  example: ["Transformer", "Attention"]
      responses:
        '201':
          description: 创建成功
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/Note'
        '400':
          description: 请求参数错误
```

### 4. 添加认证

大多数 API 需要认证，OpenSpec 支持多种方式：

```yaml
components:
  securitySchemes:
    BearerAuth:
      type: http
      scheme: bearer
      bearerFormat: JWT
    ApiKeyAuth:
      type: apiKey
      in: header
      name: X-API-Key

security:
  - BearerAuth: []
```

## 实用工具推荐

### 用 Python 验证和解析 OpenSpec

```python
# 安装依赖：pip install pyyaml openapi-spec-validator

import yaml
from openapi_spec_validator import validate

# 读取并验证 spec 文件
with open("openapi.yaml", "r") as f:
    spec = yaml.safe_load(f)

# 验证是否符合 OpenAPI 规范
validate(spec)
print("✅ OpenAPI spec 验证通过！")

# 遍历所有端点
for path, methods in spec["paths"].items():
    for method, details in methods.items():
        print(f"{method.upper()} {path} - {details.get('summary', '')}")
```

### 自动生成客户端代码

使用 `openapi-generator` 可以从 spec 文件自动生成各种语言的 SDK：

```bash
# 安装
npm install @openapitools/openapi-generator-cli -g

# 生成 Python 客户端
openapi-generator-cli generate \
  -i openapi.yaml \
  -g python \
  -o ./generated-client

# 生成 TypeScript 客户端
openapi-generator-cli generate \
  -i openapi.yaml \
  -g typescript-axios \
  -o ./generated-ts-client
```

### Swagger UI 可视化

把你的 spec 文件拖到 [Swagger Editor](https://editor.swagger.io/) 就能看到漂亮的交互式文档，还能直接发送测试请求。

## OpenSpec 在 AI 领域的应用

### 1. AI Agent 工具调用

很多 AI Agent 框架（如 LangChain、AutoGPT）使用 OpenSpec 来描述可调用的工具。Agent 读取 spec 后就能理解如何调用外部 API：

```python
# 伪代码：Agent 根据 OpenSpec 自动调用 API
agent.register_tool_from_openapi("https://api.example.com/openapi.yaml")
agent.run("帮我查找所有关于 Transformer 的学习笔记")
# Agent 自动调用 GET /notes?category=Transformer
```

### 2. MCP Server 定义

Model Context Protocol（MCP）中，工具的定义方式与 OpenSpec 的思想一脉相承——用结构化的方式描述输入输出，让 AI 模型能够理解和调用。

### 3. API 文档即代码

在 AI 微服务架构中，每个服务都可以暴露自己的 OpenSpec，实现：
- 服务发现和自动注册
- API 网关路由配置
- 自动化集成测试

## 最佳实践

1. **版本管理**：把 `openapi.yaml` 纳入 Git 管理，API 变更有迹可循
2. **先设计后编码**：API-First 开发模式，先写 spec 再实现
3. **使用 `$ref` 复用**：避免重复定义相同的 Schema
4. **写好 `example`**：每个字段都加上示例值，文档更直观
5. **保持同步**：代码变更时同步更新 spec，可以用 CI 检查一致性

## 今日小结

- **OpenSpec（OpenAPI Specification）** 是描述 RESTful API 的行业标准
- 核心组成：`info`（元信息）、`paths`（端点）、`components`（模型）
- 可以自动生成文档、客户端代码和测试
- 在 AI Agent 领域，OpenSpec 是连接 AI 模型与外部工具的桥梁
- 推荐 API-First 的开发方式：先定义 spec，再写实现代码

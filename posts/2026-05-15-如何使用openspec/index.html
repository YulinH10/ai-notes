如何使用 OpenSpec（OpenAPI Specification）：从入门到最佳实践

2026年5月15日 · 11 分钟 · 2152 字 · YulinH10

一、什么是 OpenAPI Specification？#

OpenAPI Specification（OAS），也常被称为 OpenSpec，是一种语言无关的标准接口描述格式，用于定义 HTTP API。根据 OAS 3.1.1 官方规范的定义：

The OpenAPI Specification (OAS) defines a standard, language-agnostic interface to HTTP APIs which allows both humans and computers to discover and understand the capabilities of the service without access to source code, documentation, or through network traffic inspection.

简单来说，OpenSpec 是一份机器可读、人类可理解的 API 说明书。它使用 JSON 或 YAML 格式，完整描述一个 API 的所有端点、参数、请求体、响应、认证方式等信息。

版本演进#

版本年份关键变化
Swagger 2.02014原始规范，SmartBear 公司主导
OpenAPI 3.02017正式更名为 OpenAPI，引入 components、links、多服务器支持
OpenAPI 3.12021完全兼容 JSON Schema Draft 2020-12，支持 webhooks，引入 pathItem 引用

目前推荐使用 OpenAPI 3.1.x 版本。本文所有示例均基于该版本。

二、为什么要使用 OpenSpec？#

2.1 消除沟通歧义#

在没有 OpenSpec 的团队中，API 的定义往往散落在 Confluence 文档、Slack 消息、甚至代码注释里。前后端联调时经常出现：

  • “这个字段是 string 还是 number？”
  • “这个接口的错误码有哪些？”
  • “参数是放在 query 里还是 body 里？”

OpenSpec 用一份标准化的文件解决所有这些问题。它既是文档，也是契约（Contract）。

2.2 强大的工具生态#

一旦你有了 OpenSpec 文件，整个工具链都为你服务：

工具用途代表产品
文档生成自动生成交互式 API 文档Swagger UI、Redoc、Stoplight
代码生成生成客户端 SDK 和服务端框架代码openapi-generator（支持 50+ 语言）
Mock 服务根据 spec 自动生成模拟 APIPrism、Mockoon
测试工具自动化契约测试和集成测试Schemathesis、Dredd
验证工具校验 spec 的正确性和风格Spectral、openapi-spec-validator
API 网关根据 spec 配置路由和策略Kong、AWS API Gateway

2.3 API-First 开发模式#

传统开发是先写代码再补文档（Code-First），OpenSpec 推动了 API-First 的理念：

设计 spec → 评审 spec → 前后端并行开发 → 联调验证

这种方式的优势：

  • 前端不需要等后端开发完成，可以根据 spec 先行开发
  • 后端可以用 spec 自动生成框架代码
  • API 变更必须先修改 spec，再改代码，强制保持同步

2.4 AI Agent 的 “工具说明书”#

在 AI 应用爆发的今天，OpenSpec 有了一个全新的角色——让 AI Agent 理解如何调用外部工具。

ChatGPT Plugins、LangChain、AutoGPT 等框架都使用 OpenSpec 作为 Agent 的工具描述格式。Agent 读取 spec 后，就能自动理解：

  • 有哪些 API 可以调用
  • 每个 API 需要什么参数
  • 返回什么格式的数据

这使得 OpenSpec 从 “开发者工具” 升级为 “AI 基础设施” 的关键一环。

三、如何使用 OpenSpec#

3.1 OpenAPI 文档的整体结构#

根据 OAS 3.1.1 规范，一个 OpenAPI 文档（OpenAPI Document）的根对象包含以下字段：

openapi: 3.1.1                    # 必填：规范版本号
info:                              # 必填：API 元信息
  title: My API
  version: 1.0.0
servers:                           # 服务器地址列表
  - url: https://api.example.com
paths:                             # API 端点定义
  /users:
    get: ...
webhooks:                          # 入站 Webhook 定义（3.1 新增）
  newUser:
    post: ...
components:                        # 可复用的组件
  schemas: ...
  securitySchemes: ...
security:                          # 全局安全策略
  - BearerAuth: []
tags:                              # 标签，用于分组
  - name: users
externalDocs:                      # 外部文档链接
  url: https://docs.example.com

根据规范，一个合法的 OpenAPI 文档至少需要包含 paths、components、webhooks 中的一个。

3.2 Info Object：API 的身份证#

info:
  title: AI 学习笔记 API           # 必填：API 标题
  summary: 管理 AI 学习笔记的接口    # 简短摘要
  description: |                   # 详细描述，支持 CommonMark Markdown
    这是一个用于管理 AI 学习笔记的 RESTful API。
    
    ## 功能
    - 创建、查询、更新、删除笔记
    - 按标签和分类筛选
    - 全文搜索
  version: 2.1.0                   # 必填：API 版本（非规范版本）
  contact:
    name: API Support
    email: support@example.com
    url: https://example.com/support
  license:
    name: MIT
    identifier: MIT                # 3.1 新增：SPDX 许可证标识符

注意：info.version 是你的 API 版本，不是 OpenAPI 规范版本。规范版本由顶层的 openapi 字段指定。

3.3 Server Object：定义服务器#

支持多环境和变量模板：

servers:
  - url: https://api.example.com/v1
    description: 生产环境
  - url: https://staging-api.example.com/v1
    description: 预发布环境
  - url: https://{environment}.example.com:{port}/v1
    description: 自定义环境
    variables:
      environment:
        default: api
        enum:
          - api
          - sandbox
          - staging
      port:
        default: '443'

如果 servers 字段缺失或为空数组，默认值相当于 [{url: "/"}]，即相对于文档所在的地址。

3.4 Paths Object：定义 API 端点#

这是 OpenSpec 的核心。每个路径下可以定义 get、post、put、delete、patch、head、options、trace 方法：

paths:
  /notes:
    summary: 笔记资源
    description: 对学习笔记进行 CRUD 操作

    get:
      operationId: listNotes
      summary: 获取笔记列表
      tags:
        - notes
      parameters:
        - name: category
          in: query               # 参数位置：query / header / path / cookie
          required: false
          description: 按分类筛选笔记
          schema:
            type: string
            enum: [AI基础, 深度学习, NLP, LLM, CV]
        - name: page
          in: query
          schema:
            type: integer
            minimum: 1
            default: 1
        - name: limit
          in: query
          schema:
            type: integer
            minimum: 1
            maximum: 100
            default: 20
      responses:
        '200':
          description: 成功
          content:
            application/json:
              schema:
                type: object
                properties:
                  data:
                    type: array
                    items:
                      $ref: '#/components/schemas/Note'
                  total:
                    type: integer
                    example: 42
              example:
                data:
                  - id: 1
                    title: "Transformer 学习笔记"
                    category: "深度学习"
                total: 42
        '401':
          $ref: '#/components/responses/Unauthorized'
        '500':
          $ref: '#/components/responses/InternalError'

    post:
      operationId: createNote
      summary: 创建新笔记
      tags:
        - notes
      requestBody:
        required: true
        content:
          application/json:
            schema:
              $ref: '#/components/schemas/CreateNoteRequest'
      responses:
        '201':
          description: 创建成功
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/Note'
          headers:
            Location:
              description: 新创建资源的 URL
              schema:
                type: string
        '400':
          $ref: '#/components/responses/BadRequest'
        '422':
          description: 验证失败
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/ValidationError'

  /notes/{noteId}:
    parameters:
      - name: noteId
        in: path
        required: true            # path 参数必须为 required: true
        schema:
          type: integer

    get:
      operationId: getNote
      summary: 获取单个笔记
      tags:
        - notes
      responses:
        '200':
          description: 成功
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/Note'
        '404':
          $ref: '#/components/responses/NotFound'

    put:
      operationId: updateNote
      summary: 更新笔记
      tags:
        - notes
      requestBody:
        required: true
        content:
          application/json:
            schema:
              $ref: '#/components/schemas/UpdateNoteRequest'
      responses:
        '200':
          description: 更新成功
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/Note'

    delete:
      operationId: deleteNote
      summary: 删除笔记
      tags:
        - notes
      responses:
        '204':
          description: 删除成功

3.5 Components Object：可复用的组件#

components 是 OpenSpec 中实现 DRY（Don’t Repeat Yourself）原则的关键。它可以存放可复用的 schemas、responses、parameters、requestBodies、headers、securitySchemes 等：

components:
  schemas:
    Note:
      type: object
      required:
        - id
        - title
        - content
      properties:
        id:
          type: integer
          readOnly: true
          example: 1
        title:
          type: string
          minLength: 1
          maxLength: 200
          example: "Transformer 学习笔记"
        content:
          type: string
          minLength: 1
          example: "今天学习了 Self-Attention 机制..."
        category:
          type: string
          enum: [AI基础, 深度学习, NLP, LLM, CV]
          example: "深度学习"
        tags:
          type: array
          items:
            type: string
          example: ["Transformer", "Attention"]
        created_at:
          type: string
          format: date-time
          readOnly: true
        updated_at:
          type: string
          format: date-time
          readOnly: true

    CreateNoteRequest:
      type: object
      required:
        - title
        - content
      properties:
        title:
          type: string
          minLength: 1
          maxLength: 200
        content:
          type: string
          minLength: 1
        category:
          type: string
        tags:
          type: array
          items:
            type: string

    UpdateNoteRequest:
      type: object
      properties:
        title:
          type: string
          minLength: 1
          maxLength: 200
        content:
          type: string
          minLength: 1
        category:
          type: string
        tags:
          type: array
          items:
            type: string

    Error:
      type: object
      required:
        - code
        - message
      properties:
        code:
          type: string
        message:
          type: string
        details:
          type: array
          items:
            type: object
            properties:
              field:
                type: string
              reason:
                type: string

    ValidationError:
      allOf:
        - $ref: '#/components/schemas/Error'
        - type: object
          properties:
            code:
              example: "VALIDATION_ERROR"

  responses:
    BadRequest:
      description: 请求参数错误
      content:
        application/json:
          schema:
            $ref: '#/components/schemas/Error'
          example:
            code: "BAD_REQUEST"
            message: "请求参数格式不正确"

    Unauthorized:
      description: 未认证
      content:
        application/json:
          schema:
            $ref: '#/components/schemas/Error'
          example:
            code: "UNAUTHORIZED"
            message: "请提供有效的认证凭证"

    NotFound:
      description: 资源不存在
      content:
        application/json:
          schema:
            $ref: '#/components/schemas/Error'
          example:
            code: "NOT_FOUND"
            message: "请求的资源不存在"

    InternalError:
      description: 服务器内部错误
      content:
        application/json:
          schema:
            $ref: '#/components/schemas/Error'
          example:
            code: "INTERNAL_ERROR"
            message: "服务器内部错误，请稍后重试"

  securitySchemes:
    BearerAuth:
      type: http
      scheme: bearer
      bearerFormat: JWT
      description: 使用 JWT Bearer Token 进行认证

    ApiKeyAuth:
      type: apiKey
      in: header
      name: X-API-Key
      description: 使用 API Key 进行认证

通过 $ref 关键字引用 components 中的定义，避免重复代码：

# 引用 Schema
$ref: '#/components/schemas/Note'

# 引用 Response
$ref: '#/components/responses/NotFound'

# 引用远程文件中的定义
$ref: './common/errors.yaml#/components/schemas/Error'

3.6 Security：认证与授权#

OpenSpec 支持五种安全方案类型：

components:
  securitySchemes:
    # 1. HTTP 认证（Basic、Bearer 等）
    BearerAuth:
      type: http
      scheme: bearer
      bearerFormat: JWT

    # 2. API Key（header、query 或 cookie）
    ApiKeyAuth:
      type: apiKey
      in: header
      name: X-API-Key

    # 3. OAuth 2.0
    OAuth2:
      type: oauth2
      flows:
        authorizationCode:
          authorizationUrl: https://auth.example.com/authorize
          tokenUrl: https://auth.example.com/token
          scopes:
            notes:read: 读取笔记
            notes:write: 创建和修改笔记
            notes:delete: 删除笔记

    # 4. OpenID Connect
    OpenIDConnect:
      type: openIdConnect
      openIdConnectUrl: https://auth.example.com/.well-known/openid-configuration

    # 5. 双向 TLS（3.1 新增）
    MutualTLS:
      type: mutualTLS

# 全局安全策略：所有接口默认需要 Bearer Token
security:
  - BearerAuth: []

# 也可以在单个 Operation 上覆盖
paths:
  /public/health:
    get:
      security: []                 # 空数组 = 不需要认证
      responses:
        '200':
          description: OK

3.7 OAS 3.1 的数据类型#

OAS 3.1 的 Schema Object 完全兼容 JSON Schema Draft 2020-12。支持的基础类型：

JSON 类型OAS format 扩展说明
stringdate-time, date, email, uri, uuid, password文本数据
numberfloat, double浮点数
integerint32, int64整数（JSON 中本质是 number）
boolean—布尔值
array—数组，需配合 items
object—对象，需配合 properties
null—空值（3.1 新增，可与其他类型组合）

3.1 中可以用数组语法表示可空类型，替代 3.0 中的 nullable: true：

# OAS 3.0 的写法
type: string
nullable: true

# OAS 3.1 的写法（推荐）
type:
  - string
  - "null"

四、实用工具链#

4.1 验证 Spec 的正确性#

# pip install pyyaml openapi-spec-validator

import yaml
from openapi_spec_validator import validate

with open("openapi.yaml", "r", encoding="utf-8") as f:
    spec = yaml.safe_load(f)

validate(spec)
print("OpenAPI spec 验证通过")

# 遍历所有端点
for path, path_item in spec.get("paths", {}).items():
    for method in ["get", "post", "put", "delete", "patch"]:
        if method in path_item:
            op = path_item[method]
            op_id = op.get("operationId", "N/A")
            summary = op.get("summary", "")
            print(f"  {method.upper():6s} {path:30s} [{op_id}] {summary}")

4.2 使用 Spectral 进行风格检查#

Spectral 是 Stoplight 出品的 OpenAPI Linter，可以检查 spec 是否符合团队规范：

npm install -g @stoplight/spectral-cli

# 使用内置规则集检查
spectral lint openapi.yaml

# 使用自定义规则
spectral lint openapi.yaml --ruleset .spectral.yaml

自定义规则示例（.spectral.yaml）：

extends:
  - spectral:oas

rules:
  operation-operationId:
    severity: error               # operationId 必须存在
  oas3-valid-media-example:
    severity: warn                # example 应该符合 schema
  info-contact:
    severity: error               # 必须提供联系信息
  operation-description:
    severity: warn                # 每个操作应该有描述

五、OpenSpec 在 AI 领域的应用#

5.1 ChatGPT / GPTs 的 Actions#

OpenAI 的 GPTs 使用 OpenSpec 定义 Actions。你只需提供一份 OpenAPI spec，GPT 就能理解你的 API 并在对话中自动调用：

openapi: 3.1.0
info:
  title: Note Search API
  version: 1.0.0
servers:
  - url: https://api.example.com
paths:
  /search:
    get:
      operationId: searchNotes
      summary: 搜索学习笔记
      description: 根据关键词在笔记库中进行语义搜索
      parameters:
        - name: q
          in: query
          required: true
          schema:
            type: string
          description: 搜索关键词
      responses:
        '200':
          description: 搜索结果
          content:
            application/json:
              schema:
                type: array
                items:
                  type: object
                  properties:
                    title:
                      type: string
                    snippet:
                      type: string
                    relevance:
                      type: number

5.3 MCP（Model Context Protocol）#

MCP 的 Tool 定义思想与 OpenSpec 一脉相承——都是用结构化的 JSON Schema 来描述工具的输入输出，让 AI 理解如何调用。许多 MCP Server 的实现也支持直接从 OpenAPI spec 自动生成 Tool 定义。

六、最佳实践#

6.1 设计原则#

1. API-First：先 spec 后代码

产品需求 → 设计 OpenAPI spec → 团队评审 → 前后端并行开发 → 契约测试

不要等后端写完了再反过来生成 spec。API 设计应该是前后端共同参与的协作过程。

2. 使用语义化版本号

info:
  version: 2.1.0    # major.minor.patch
  • major：破坏性变更（删除字段、修改类型）
  • minor：向后兼容的新增（新增端点、新增可选字段）
  • patch：bug 修复，不影响 API 行为

3. 每个 Operation 必须有 operationId

operationId 是操作的唯一标识符，被代码生成器用作函数名、被 AI Agent 用作工具标识。命名建议使用 camelCase：

operationId: listNotes      # 好
operationId: get_all_notes   # 不推荐
operationId: GetNotes        # 不推荐

6.2 Schema 设计#

4. 充分利用 $ref 复用

# 不好：重复定义
paths:
  /notes:
    get:
      responses:
        '404':
          description: 未找到
          content:
            application/json:
              schema:
                type: object
                properties:
                  code: { type: string }
                  message: { type: string }
  /users:
    get:
      responses:
        '404':
          description: 未找到
          content:
            application/json:
              schema:
                type: object
                properties:
                  code: { type: string }
                  message: { type: string }

# 好：使用 $ref 复用
paths:
  /notes:
    get:
      responses:
        '404':
          $ref: '#/components/responses/NotFound'
  /users:
    get:
      responses:
        '404':
          $ref: '#/components/responses/NotFound'

5. 区分读写模型

使用 readOnly 和 writeOnly 标记字段，或者使用 allOf 组合读写模型：

components:
  schemas:
    NoteBase:
      type: object
      properties:
        title:
          type: string
        content:
          type: string

    Note:
      allOf:
        - $ref: '#/components/schemas/NoteBase'
        - type: object
          properties:
            id:
              type: integer
              readOnly: true
            created_at:
              type: string
              format: date-time
              readOnly: true

6. 给每个字段提供 example

Example 会被 Swagger UI 展示、被 Mock 工具使用、被 AI Agent 参考：

title:
  type: string
  minLength: 1
  maxLength: 200
  example: "Transformer 架构详解：从 Self-Attention 到多头注意力"

  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm install -g @stoplight/spectral-cli
      - run: spectral lint openapi/openapi.yaml --fail-severity warn

9. 拆分大型 spec 文件

单个 spec 文件超过 500 行就该考虑拆分。使用 $ref 引用外部文件：

# openapi.yaml（主文件）
paths:
  /notes:
    $ref: './paths/notes.yaml#/notes'
  /notes/{noteId}:
    $ref: './paths/notes.yaml#/notes~1{noteId}'

components:
  schemas:
    Note:
      $ref: './components/schemas.yaml#/Note'

10. 使用 tags 组织 API

tags:
  - name: notes
    description: 学习笔记管理
  - name: users
    description: 用户管理
  - name: search
    description: 搜索相关接口

Tags 在 Swagger UI 中会作为分组标题展示，帮助用户快速找到需要的接口。

info:
  title: AI 学习笔记 API
  summary: 管理 AI 学习笔记的 RESTful API
  description: |
    提供学习笔记的 CRUD 操作和搜索功能。
    支持按分类、标签进行筛选。
  version: 1.0.0
  contact:
    name: YulinH10
    url: https://github.com/YulinH10

    name: MIT
    identifier: MIT

servers:
  - url: https://api.ainotes.example.com/v1
    description: 生产环境
  - url: http://localhost:3000/v1
    description: 本地开发

  - name: notes
    description: 学习笔记 CRUD
  - name: search
    description: 全文搜索

security:
  - BearerAuth: []

paths:
  /notes:
    get:
      operationId: listNotes
      summary: 获取笔记列表
      tags: [notes]
      parameters:
        - name: category
          in: query
          schema:
            type: string
        - name: tag
          in: query
          schema:
            type: string
        - name: page
          in: query
          schema:
            type: integer
            default: 1
        - name: limit
          in: query
          schema:
            type: integer
            default: 20
            maximum: 100
      responses:
        '200':
          description: 成功
          content:
            application/json:
              schema:
                type: object
                properties:
                  data:
                    type: array
                    items:
                      $ref: '#/components/schemas/Note'
                  total:
                    type: integer
    post:
      operationId: createNote
      summary: 创建笔记
      tags: [notes]
      requestBody:
        required: true
        content:
          application/json:
            schema:
              $ref: '#/components/schemas/CreateNoteRequest'
      responses:
        '201':
          description: 创建成功
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/Note'
        '422':
          description: 验证失败
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/Error'

  /notes/{noteId}:
    parameters:
      - name: noteId
        in: path
        required: true
        schema:
          type: integer
    get:
      operationId: getNote
      summary: 获取单个笔记
      tags: [notes]
      responses:
        '200':
          description: 成功
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/Note'
        '404':
          $ref: '#/components/responses/NotFound'
    delete:
      operationId: deleteNote
      summary: 删除笔记
      tags: [notes]
      responses:
        '204':
          description: 删除成功

  /search:
    get:
      operationId: searchNotes
      summary: 搜索笔记
      tags: [search]
      parameters:
        - name: q
          in: query
          required: true
          schema:
            type: string
      responses:
        '200':
          description: 搜索结果
          content:
            application/json:
              schema:
                type: array
                items:
                  $ref: '#/components/schemas/Note'

components:
  schemas:
    Note:
      type: object
      required: [id, title, content]
      properties:
        id:
          type: integer
          readOnly: true
          example: 1
        title:
          type: string
          example: "Transformer 学习笔记"
        content:
          type: string
          example: "Self-Attention 的核心思想是..."
        category:
          type: string
          example: "深度学习"
        tags:
          type: array
          items:
            type: string
          example: ["Transformer", "Attention"]
        created_at:
          type: string
          format: date-time
          readOnly: true
        updated_at:
          type: string
          format: date-time
          readOnly: true

    CreateNoteRequest:
      type: object
      required: [title, content]
      properties:
        title:
          type: string
          minLength: 1
          maxLength: 200
        content:
          type: string
          minLength: 1
        category:
          type: string
        tags:
          type: array
          items:
            type: string

    Error:
      type: object
      required: [code, message]
      properties:
        code:
          type: string
          example: "NOT_FOUND"
        message:
          type: string
          example: "请求的资源不存在"

  responses:
    NotFound:
      description: 资源不存在
      content:
        application/json:
          schema:
            $ref: '#/components/schemas/Error'

  securitySchemes:
    BearerAuth:
      type: http
      scheme: bearer
      bearerFormat: JWT
